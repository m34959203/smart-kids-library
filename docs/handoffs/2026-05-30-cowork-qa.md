## P1 — мажорные

### P1-1. ИИ-консультант галлюцинирует — выдаёт несуществующие книги

- **Где:** `/ru/catalog/read/944` → floating-bubble «Библиотекарь-помощник».
- **Шаги:** открыть виджет → «Какие книги про Сатпаев есть в каталоге?».
- **Факт:** ответ:
  > «В нашей библиотеке есть несколько книг о Сатпаеве и его истории.
  > Одна из них — **«История города Сатпаев»** ... Ещё есть книга
  > **«Сатпаев: город горняков»** — она рассказывает о горнодобывающей
  > промышленности...»
- **Реальность:** в каталоге таких книг нет. Видимые книги о Сатпаеве —
  «АКАДЕМИК Қ.СӘТБАЕВТЫҢ ТУҒАНЫНА — 125 ЖЫЛ», «Шарайна» серия, и т.п.
- **Причина-кандидат:** Groq LLM отвечает «из головы», без реального
  RAG-поиска по `books`-таблице. Системный промпт упоминает каталог, но
  не делает поиск/embedding.
- **Команда для Code:** добавить tool `searchCatalog(query)` или
  pre-prompt с top-K результатами MiniSearch/Postgres FTS перед
  передачей в LLM. Запретить отвечать о книгах, не вернувшихся из tool.
  Альтернатива — отвечать «не нашёл, попробуйте поиск /catalog».

### P1-2. `/ru/kids/voice` → дефолтный Next.js 404 без брендинга

- **Шаги:** перейти на `/ru/kids/voice` (или `/kk/kids/voice`).
- **Факт:** чёрный фон + английский текст «404 / This page could not be
  found.» без хедера, локализации, кнопки «На главную».
- **Кроме того:** в bottom mobile-nav есть таб «Дауыс» (Голос на KK),
  но соответствующего публичного роута я не нашёл — не ясно куда этот
  таб ведёт. Если ведёт на `/kids/voice` — двойной баг.
- **Команда для Code:** (а) сделать кастомный `not-found.tsx` с
  локализацией; (б) либо реализовать `/kids/voice` (если он
  планировался), либо переделать таб «Дауыс» на ссылку на существующий
  голосовой эндпоинт (видимо, на чат-виджет с STT).

### P1-3. Регистрация на прошедшие события не блокируется

- **Шаги:** `/ru/events` → клик на «Мастер-класс по книжному дизайну»
  (10 апреля 2026, прошедшее) → `/ru/events/9`.
- **Факт:** карточка показывает кнопку **«Записаться»**, нет метки
  «Прошло». Никакого banner про прошедшее событие.
- **На календаре** сегодня (30 мая) выделено зелёным, но в списке справа
  показаны 9/14/18/22/28 мая — все прошедшие — без отметок «прошло».
  Только последнее «31 мая, Книжная выставка» имеет бейдж
  «1 дн. осталось» (отлично!).
- **Команда для Code:**
  - на детале: если `end_date < now` → скрыть форму регистрации +
    показать сообщение «Регистрация на это событие завершена» (этот
    паттерн уже реализован в smart-library-cbs).
  - на списке: сортировать upcoming first, прошедшие в свёрнутую секцию
    или серым ghost-стилем + бейдж «Прошло» / «Өткен».
  - на бэкенде POST `/api/events/register` должен возвращать
    `409 EVENT_ENDED`, если событие прошло — защита от XHR-обхода.

### P1-4. TTS зависает в «Готовлю...» (читалка + сказки)

- **Шаги 1:** `/ru/catalog/read/944` → кнопка «🔊 Слушать» → после клика
  кнопка показывает «Готовлю...» и не возвращается ни в `Играть`, ни в
  ошибку, ни не воспроизводится. Ждал 25 секунд.
- **Шаги 2:** `/ru/kids/stories` → сгенерировал сказку → кнопка
  «Послушать» → та же история: «Готовлю…» зависает.
- **Network**: запросы к TTS-эндпоинту были (по индикатору кнопки),
  но не успел перехватить URL до фриза.
- **Причина-кандидат:** Gemini KK-TTS квота, или ElevenLabs/GCloud
  ru-TTS не настроены, или fallback застрял. По README — основной
  TTS-стек это Gemini для KK, ElevenLabs/GCloud для RU. Если RU-ключи
  не заданы → silent fail без timeout.
- **Команда для Code:** добавить таймаут на TTS (~15 c) с понятным
  toast «Не удалось озвучить, попробуйте позже» и логировать причину
  (quota / no_key / network) в `ai_generations` журнал.

## P2 — заметные UX-регрессии

### P2-1. Часть карточек каталога с «невидимыми» обложками (низкий контраст)

- **Где:** `/ru/catalog`, верхняя секция «Из коллекции» — первые ~6
  карточек (Шарайна №2, КҮЗДЕ БАСТАЛҒАН, ШЫРАҒЫҢ СӨНБЕСІҢ, ҰРПАҚҚА
  ҰЛАН БОЛҒАН ҰЛЫ, «ЗАҢ мен ТӘРТІП» — ЖАСТАР).
- **Факт:** только пастельный фон (бежевый/мятный/голубой) без видимого
  текста. Дальше по списку все карточки уже коричневые с белым текстом
  (типографические SVG), читается хорошо.
- **Возможно:** обложки этих 6 карточек сгенерированы с очень светлым
  font-color или вообще пустые. README заявляет 100% покрытие, но
  визуально первые 6 «не видно».
- **Команда для Code:** проверить `cover_url` у этих 6 записей через
  `SELECT id, title, cover_url FROM books LIMIT 6`. Если URL пустой —
  fallback на типографику. Если URL ведёт на SVG со светлым font —
  пересгенерировать.

### P2-2. Прошедшие события в списке без фильтрации/маркера

- См. P1-3 — на списке `/ru/events` смешаны прошлые и будущие, нет
  раздела «Прошедшие» / сортировки upcoming-first.

### P2-3. «Войти» через `/profile/login` → 404

- **Шаги:** прямой URL `/ru/profile/login` или `/kk/profile/login`.
- **Факт:** h1=«404». Логин-форма живёт на `/profile` (с
  `?next=/...`-параметром при попытке зайти в защищённый ресурс).
- **Если** в UI где-то есть статичная ссылка «Войти» → её надо вести на
  `/profile`. Либо завести шим `/profile/login` → redirect на `/profile`.

### P2-4. Header brand-mark: невидимая аватарка-плейсхолдер

- Между LIVE-индикатором и возрастными чипами висит **пустой круг
  бежевого цвета**, нет alt/title — не понятно, аватарка пользователя
  или плейсхолдер библиотекаря.
- **Команда для Code:** убрать (если placeholder) или поставить логотип
  библиотеки.

### P2-5. KK vs RU несоответствие в nav: «Балаларға» vs «Балалар»

- В header (`/kk`): «Балаларға» (родит. падеж, «для детей»).
- В bottom mobile-tab: «Балалар» (именит. падеж, «дети»).
- Оба варианта приемлемые, но **в одном UI быть должны одинаковыми**.

## P3 — мелочи

- **P3-1.** Hero h1: «Оқу әлеміне қош келдіңіз .» — точка через
  не-разрывный гэп от текста (видна как «отдельно стоящая» на новой
  строке). RU аналогично «Библиотека, которую любят дети.» — на RU
  плотно, на KK — отвалилась. Перевернуть HTML-точку в общий span.
- **P3-2.** Возрастные карточки на главной (`/ru` пастельные блоки) —
  №2 «10-13» и №3 «14-17» **без иллюстраций**, только цветной фон с
  заголовком. №1 «6-9» имеет иллюстрацию. Доложить иллюстрации.
- **P3-3.** На detail-странице книги `/ru/catalog/944` под обложкой
  стоит «0» без подписи — непонятно (тираж? просмотры? страниц?).
  Добавить label или скрыть, если 0.
- **P3-4.** В reader: «Прямая ссылка: /uploads/books/book_0926.docx»
  отрендерена как plain text, не кликабельный `<a href>`. И не
  обязательно показывать обычному читателю.
- **P3-5.** Снеки/snackbar «Выберите возрастную группу...» и «Что
  почитать? Спроси меня» внизу — два разных мини-виджета поверх контента,
  частично пересекаются с floating-bubble. На мобиле может быть тесно.
- **P3-6.** В AI-ответе hyphen-minus (`-`) вместо em/en-dash и
  straight-quotes (`"..."`) вместо умных кавычек (`«...»`) —
  типографически беднее остального UI.

## Что НЕ проверено (требует доступа / времени)

- **Админка** (включая автопостинг IG/TG) — нужен admin-логин/пароль.
- **STT-диктовка** (микрофон в чате) — Web Speech API в Chrome требует
  permission на mic в remote-сессии, не активировал.
- **TTS реального аудио** — кнопки зависают, см. P1-4.
- **/kids/voice** или эквивалент голосового помощника — URL не нашёл,
  страница 404.
- **Полный flow восстановления пароля** через email-link (`/profile/reset`)
  — нужно сначала запустить recover с реальным email и поймать токен.
- **Викторины** `/kids/quizzes`, **мастерская** `/kids/workshop`,
  **раскраски** `/kids/coloring` — не углублялся, время.
- **Новости** `/ru/news` — не открывал.
- **Lighthouse / accessibility deep scan** — не запускал.
- **Cross-browser** — только Chrome 148.

## Решения по фиксам (Дмитрий, 2026-05-30) + уточнения верификации

**P1-2 уточнён.** Таб «Голос/Дауыс» в `BottomNav.tsx` ведёт на `/live`
(`labelKey:"live"`), и `/live` отвечает **200** (Gemini Live —

  НЕ Gemini Live. Чат-STT уже есть в `ChatWidget` (`startVoice`,
  `SpeechRecognition`, mic-кнопка). Нужно: либо удалить осиротевший
  `/kids/voice` и перенаправить карточку хаба на открытие чат-виджета со
  STT, либо сделать `/kids/voice` тонкой страницей, монтирующей этот же
  чат в voice-режиме. Никаких обещаний фич, которых нет.

полнотекстовый поиск по `books` (Postgres уже есть), top-K в системный

откройте /catalog». Без новых зависимостей и без tool-calling.
